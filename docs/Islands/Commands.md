# Islands 插件命令参考

## 📋 命令总览

### 主命令
```
/islands [子命令]
/is [子命令]
```

### 命令分类
- 🏠 **岛屿管理**: 创建、删除、设置
- 👥 **团队管理**: 邀请、踢出、离开
- 🛡️ **权限控制**: 锁定、PVP、物品控制
- 📊 **信息查询**: 帮助、模板、状态
- ⚙️ **系统管理**: 重载、调试

---

## 🏠 岛屿管理命令

### 🏝️ 岛屿管理

#### `/islands create [模板名称]`
创建你的空岛世界

**参数：**
- `模板名称` (可选) - 指定要使用的岛屿模板

**权限：** `islands.create` (默认拥有)

**示例：**
```
/islands create
/islands create a
/islands create default
```

**说明：**
- 创建成功后会自动建立团队并设置世界边界
- 支持多模板系统，可以创建不同类型的岛屿
- 使用 `islands.templates` 指令查看可用模板

---

#### `/islands remove`
删除你的岛屿世界

**权限：** `islands.remove` (默认拥有)

**说明：**
- 需要二次确认，执行后60秒内使用 `/islands remove confirm` 确认
- 删除操作不可逆，请谨慎使用
- 有删除次数限制，可在配置中设置

#### `/islands remove confirm`
确认删除岛屿世界

**权限：** `islands.remove` (默认拥有)

**说明：**
- 必须在执行 `/islands remove` 后60秒内使用
- 确认后立即删除岛屿世界

---

#### `/islands sethome`
设置岛屿世界出生点

**权限：** `islands.sethome` (默认拥有)

**说明：**
- 只能在你的岛屿世界中使用
- 设置后死亡会在此位置复活

---

### 👥 团队管理

#### `/islands invite <玩家名>`
邀请玩家加入你的岛屿团队

**参数：**
- `玩家名` - 要邀请的玩家名称

**权限：** `islands.invite` (默认拥有)

**示例：**
```
/islands invite Steve
/islands invite Alex
```

**说明：**
- 只有岛主可以邀请玩家
- 邀请有效期为60秒（可配置）
- 被邀请玩家需要使用 `/islands accept` 接受邀请

#### `/islands accept`
接受岛屿团队邀请

**权限：** `islands.invite` (默认拥有)

**说明：**
- 接受当前待处理的邀请
- 加入团队后可以进入岛屿世界

#### `/islands reject`
拒绝岛屿团队邀请

**权限：** `islands.invite` (默认拥有)

**说明：**
- 拒绝当前待处理的邀请
- 拒绝后邀请失效

#### `/islands kick <玩家名>`
踢出团队成员

**参数：**
- `玩家名` - 要踢出的玩家名称

**权限：** `islands.kick` (默认拥有)

**示例：**
```
/islands kick Steve
```

**说明：**
- 只有岛主可以踢出成员
- 不能踢出自己
- 被踢出的玩家会被传送到主世界

#### `/islands leave`
离开当前团队

**权限：** `islands.leave` (默认拥有)

**说明：**
- 岛主不能直接离开，需要先删除岛屿
- 离开后无法再进入该岛屿世界

### 🔧 岛屿设置

#### `/islands lock`
锁定/解锁岛屿世界

**权限：** `islands.lock` (默认拥有)

**说明：**
- 锁定后只有团队成员可以进入
- 其他玩家无法进入锁定的岛屿世界
- 拥有 `islands.bypass` 权限的玩家可以绕过限制

#### `/islands border`
开启/关闭世界边界

**权限：** `islands.border` (默认拥有)

**说明：**
- 仅对当前团队成员生效
- 关闭边界后团队成员可以自由移动
- 边界范围可在配置中设置

#### `/islands pvp`
开启/关闭PVP

**权限：** `islands.pvp` (默认拥有)

**说明：**
- 仅对非团队成员生效
- 团队成员之间不会受到PVP影响
- 可以控制访客之间的战斗

#### `/islands drop`
开启/关闭物品丢弃

**权限：** `islands.drop` (默认拥有)

**说明：**
- 仅对非团队成员生效
- 控制访客是否可以丢弃物品

#### `/islands pickup`
开启/关闭物品拾取

**权限：** `islands.pickup` (默认拥有)

**说明：**
- 仅对非团队成员生效
- 控制访客是否可以拾取物品

---

### 🌍 世界功能

#### `/islands expel <玩家名>`
踢出岛屿世界的访客

**参数：**
- `玩家名` - 要踢出的玩家名称

**权限：** `islands.expel` (默认拥有)

**示例：**
```
/islands expel Visitor
```

**说明：**
- 只有岛主可以踢出访客
- 不能踢出团队成员
- 被踢出的玩家会被传送到主世界

#### `/islands biomes`
打开生物群系选择界面

**权限：** `islands.biomes` (默认拥有)

**说明：**
- 打开GUI界面选择岛屿生物群系
- 可以改变岛屿世界的环境类型

---

**说明：**

## 管理员指令

### 🔧 系统管理

#### `/islands team setmax <岛主名> <最大成员数>`
设置团队最大成员数

**参数：**
- `岛主名` - 目标岛主的玩家名称
- `最大成员数` - 新的最大成员数

**权限：** `islands.team.setmax` (仅管理员)

**示例：**
```
/islands team setmax Steve 10
/islands team setmax Alex 5
```

**说明：**
- 只有管理员（OP）可以使用
- 可以设置任何岛主的团队最大成员数
- 最大成员数不能小于1
- 当前成员数不能超过新的最大值

#### `/islands reload`
重载插件配置

**权限：** `islands.reload` (仅管理员)

**说明：**
- 重载所有配置文件
- 包括 config.yml、messages.yml 等
- 修改配置后需要重载生效

## 帮助指令

### `/islands help [页码]`
查看指令帮助

**参数：**
- `页码` (可选) - 指定要查看的帮助页面

**权限：** `islands.help` (默认拥有)

**示例：**
```
/islands help
/islands help 2
```

**说明：**
- 显示所有可用指令的列表
- 支持分页显示
- 每页显示6条指令（可配置）
