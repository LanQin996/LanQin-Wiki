# Islands 插件命令参考

## 📋 命令总览

### 主命令
```
/islands [子命令]
/is [子命令]
```

### 命令分类
- 🏠 **岛屿管理**: 创建、删除、设置
- 👥 **团队管理**: 邀请、踢出、离开
- 🛡️ **权限控制**: 锁定、PVP、物品控制
- 📊 **信息查询**: 帮助、模板、状态
- ⚙️ **系统管理**: 重载、调试

---

## 🏠 岛屿管理命令

### 创建空岛
```
/islands create [模板名称]
```

**功能**: 创建新的空岛世界和团队

**参数**:
- `模板名称` (可选): 指定使用的空岛模板

**权限**: `islands.create` (默认: true)

**示例**:
```bash
/islands create          # 使用默认模板
/islands create a        # 使用模板a
/islands create premium  # 使用premium模板
```

**成功消息**: 
- 默认模板: `&a空岛创建成功！已为你建立团队并设置世界边界。`
- 指定模板: `&a使用模板 '模板名' 创建空岛成功！已为你建立团队并设置世界边界。`

**失败情况**:
- 已有空岛: 提示已有空岛
- 权限不足: `&c你没有权限执行该操作！`
- 模板不存在: 提示模板不存在

---

### 删除空岛
```
/islands remove [confirm]
```

**功能**: 删除自己的空岛世界和团队

**参数**:
- `confirm` (必需): 确认删除操作

**权限**: `islands.remove` (默认: true)

**使用流程**:
1. 执行 `/islands remove` - 开始删除流程
2. 60秒内执行 `/islands remove confirm` - 确认删除

**示例**:
```bash
/islands remove          # 开始删除流程
/islands remove confirm  # 确认删除
```

**消息**:
- 开始删除: `&e警告：你确定要删除你的岛屿世界吗？删除操作不可逆！\n&e请在60秒内执行 &c/islands remove confirm &e来确认删除。`
- 确认成功: `&a岛屿世界已成功删除！你还可以删除 X 次。`
- 超时: `&c删除岛屿世界操作已超时，取消删除。`
- 无待确认: `&c你没有任何待确认的删除操作！`

**限制**:
- 删除次数限制: 默认3次 (可在config.yml中配置)
- 达到限制: `&c你已达到最大删除岛屿次数（X次），无法再删除！`

---

### 设置出生点
```
/islands sethome
```

**功能**: 设置空岛世界的出生点

**权限**: `islands.sethome` (默认: true)

**要求**:
- 必须是空岛岛主
- 必须在自己的空岛世界内

**示例**:
```bash
/islands sethome
```

**消息**:
- 成功: `&a已将你的空岛出生点设置为当前位置！`
- 无岛屿: `&c你没有岛屿,无法设置出生点！`
- 错误世界: `&c你只能在自己的岛屿世界设置出生点！`

## 👥 团队管理命令

### 邀请成员
```
/islands invite <玩家名>
```

**功能**: 邀请玩家加入空岛团队

**参数**:
- `玩家名` (必需): 要邀请的玩家名称

**权限**: `islands.invite` (默认: true)

**示例**:
```bash
/islands invite Steve
/islands invite Alex
```

**消息**:
- 成功: `&a你已邀请 &e玩家名 &a加入你的岛屿团队!`
- 目标收到: `&e你的名字 &a邀请你加入他的岛屿！(有效期60s)`
- 不能邀请自己: `&c不能邀请自己!`
- 目标已有团队: `&c玩家名 已有团队，无法邀请!`
- 玩家不存在: `&c找不到该玩家！`
- 玩家不在线: `&c玩家不在线!`

---

### 接受邀请
```
/islands accept
```

**功能**: 接受团队邀请

**权限**: `islands.invite` (默认: true)

**要求**: 必须有待处理的邀请

**示例**:
```bash
/islands accept
```

**消息**:
- 成功: `&a你已成功加入 &e岛主名 &a的岛屿团队！`
- 岛主收到: `&e玩家名 &a已加入你的岛屿团队！`
- 无邀请: `&c你没有待处理的邀请！`
- 邀请者离线: `&c邀请者已离线，邀请失效！`
- 邀请者非岛主: `&c邀请者已失去岛主身份，邀请失效！`

---

### 拒绝邀请
```
/islands reject
```

**功能**: 拒绝团队邀请

**权限**: `islands.invite` (默认: true)

**要求**: 必须有待处理的邀请

**示例**:
```bash
/islands reject
```

**消息**:
- 成功: `&c你已拒绝岛屿团队邀请`
- 邀请者收到: `&e玩家名 &c拒绝了你的岛屿团队邀请`
- 无邀请: `&c你没有待处理的邀请！`

---

### 踢出成员

**功能**: 踢出团队成员

**参数**:
- `玩家名` (必需): 要踢出的玩家名称

**权限**: `islands.kick` (默认: true)

**要求**: 必须是岛主

**示例**:
```bash

```

- 不是岛主: `&c只有岛主才能进行此操作！`
- 不能踢自己: `&c你不能踢出自己！`
- 不是成员: `&c该玩家不是你的团队成员！`
- 玩家不存在: `&c找不到该玩家！`

---

### 离开团队
```
/islands leave
```

**功能**: 离开当前团队

**权限**: `islands.leave` (默认: true)

**要求**: 不能是岛主

**示例**:
```bash

```

**消息**:
- 成功: `&a你已成功退出团队！`
- 是岛主: `&c你是岛主，不能直接退出团队！如需解散请使用 &e/is remove`
- 无团队: `&c你没有加入任何团队！`

## ⚠️ 错误处理

### 常见错误消息

| 错误类型 | 消息 | 解决方案 |
|----------|------|----------|
| 权限不足 | `&c你没有权限执行该操作！` | 检查权限节点 |
| 玩家不存在 | `&c找不到该玩家！` | 检查玩家名称拼写 |
| 玩家不在线 | `&c玩家不在线!` | 等待玩家上线 |
| 无空岛 | `&c你还没有空岛，请先创建！` | 先创建空岛 |
| 不是岛主 | `&c只有岛主才能进行此操作！` | 需要岛主权限 |
| 已有团队 | `&c玩家名 已有团队，无法邀请!` | 目标已有团队 |
| 无团队 | `&c你没有加入任何团队！` | 需要先加入团队 |

### 命令使用提示

当输入不完整的命令时，系统会提示可用的子命令：

```
§e可用子命令: §acreate§e, §ainvite§e, §akick§e, §aleave
```

## 📝 命令使用技巧

### 1. 快速操作
- 使用 `/is` 代替 `/islands` 节省输入
- 利用Tab补全功能快速输入命令

### 2. 权限管理
- 在config.yml中设置默认权限
- 使用权限插件进行细粒度控制

### 3. 团队管理
- 定期清理不活跃成员
- 合理设置岛屿权限

### 4. 安全操作
- 删除空岛前确认重要数据已备份
- 谨慎使用踢出功能
